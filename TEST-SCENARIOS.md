## Milestone 1 — Centers rename + Task Inbox

### S1.1 — Rename shows everywhere / التسمية تظهر في كل مكان
- [ ] **خطوات:** سجّل دخول → افتح القائمة الجانبية → افتح صفحة "المراكز".
- [ ] **متوقع:** القائمة الجانبية تقول **"المراكز"** (مو "المجلات")؛ عنوان الصفحة "المراكز"؛ زر الإضافة "مركز جديد".
- [ ] **بالإنجليزي:** بدّل اللغة لـ English → نفس الأماكن تقول **"Centers" / "New Center"**.

### S1.2 — Create a center toast / رسالة إنشاء مركز
- [ ] **خطوات:** في صفحة المراكز، أنشئ مركز جديد واحفظ.
- [ ] **متوقع:** تطلع رسالة **"تم إنشاء المركز"** (EN: "Center created") — مو "المجلة".

### S1.3 — Inbox is the landing page / صندوق المهام هو صفحة البداية
- [ ] **خطوات:** سجّل دخول من جديد.
- [ ] **متوقع:** تنزل مباشرة على **"صندوق المهام"** (مو لوحة التحكم القديمة).
- [ ] لوحة التحكم القديمة لا زالت موجودة وتُفتح من القائمة الجانبية.

### S1.4 — Inbox sections show the right data / أقسام الصندوق تعرض الصح
- [ ] **طلبات تحكيم بانتظار ردك:** بحساب محكّم عنده طلب تحكيم جديد (status=pending) → يظهر القسم، والضغط يوديه لصفحة التحكيم.
- [ ] **أبحاث تحت تحكيمك:** محكّم قبِل تحكيم (status=accepted) → يظهر القسم، الضغط يفتح نموذج التقرير.
- [ ] **أبحاثي التي تحتاج تعديلاً:** باحث عنده بحث (status=revision_required) → يظهر، الضغط يفتح البحث.
- [ ] **إشعارات غير مقروءة:** حساب عنده إشعارات غير مقروءة → تظهر (أحدث 5)، و"عرض الكل" يفتح صفحة الإشعارات.

### S1.5 — Empty states / الحالات الفارغة
- [ ] **خطوات:** سجّل دخول بحساب ما عنده أي مهام معلّقة.
- [ ] **متوقع:** كل قسم فاضي **يختفي**، ويظهر بدالهم: **"لا توجد مهام معلقة حالياً 🎉"**.

### S1.6 — Count badges / شارات العدد
- [ ] **متوقع:** كل قسم فيه عناصر يعرض **شارة بعددها** الصحيح.

### S1.7 — RTL/LTR / اتجاه الصفحة
- [ ] عربي → الصفحة من اليمين لليسار (RTL). إنجليزي → من اليسار لليمين (LTR).

## Milestone 2 — Center walls

> ⚠️ هذي الاختبارات تنعمل على **قاعدة البيانات الحيّة بعد تطبيق الـ migrations**، مو في المعاينة.
> جهّز 3 حسابات تجريبية: **U_A** (عضو مركز A فقط)، **U_B** (عضو مركز B فقط)، **U_HQ** (مدير مقر `hq_admin`).

### S2.1 — Center A can't see Center B's papers / الجدار الأساسي
- [ ] سجّل دخول بـ **U_A** → افتح صفحة الأبحاث.
- [ ] **متوقع:** يشوف أبحاث مركز A **فقط**، وما يظهر له ولا بحث من مركز B.
- [ ] بـ **U_B** العكس (يشوف B فقط).

### S2.2 — HQ admin sees everything / مدير المقر يشوف الكل
- [ ] سجّل دخول بـ **U_HQ** → افتح الأبحاث/اللجان/المالية.
- [ ] **متوقع:** يشوف بيانات **كل المراكز** مجمّعة.

### S2.3 — Direct access blocked / المنع حتى بالرابط المباشر
- [ ] بـ **U_A**، حاول تفتح بحث من مركز B عن طريق رابطه المباشر (id).
- [ ] **متوقع:** ما يفتح / ما فيه بيانات (مو بس مخفي من القائمة).

### S2.4 — Walls on all data types / الجدران على كل أنواع البيانات
- [ ] بـ **U_A**، تأكد إنه يشوف بس بيانات مركزه في: اللجان، المالية، القائمة السوداء، نوافذ التقديم، معايير التقييم، الرسائل، تقارير التحكيم.

### S2.5 — No lock-out / ما فيه أحد منقفل
- [ ] بعد التطبيق، سجّل دخول بحسابات حقيقية موجودة → تأكد إنهم يشوفون بياناتهم (مو شاشة فاضية).
- [ ] **خاص:** محرّر عام (editor) ما عنده أبحاث/لجان → لو طلعت له شاشة فاضية، أضف له عضوية يدوياً أو `hq_admin`.

### S2.6 — Write walls / منع الكتابة عبر المراكز
- [ ] بـ **U_A**، حاول تنشئ بحث/لجنة وتربطه بمركز B.
- [ ] **متوقع:** يُرفض (ما يسمح).

### S2.7 — No regression / ما خربنا شي قديم
- [ ] باحث يشوف أبحاثه هو. محكّم يشوف الأبحاث المسندة له. الإشعارات تظهر زي قبل.

## Milestone 3 — Committee voting + stage tracking

### Feature 2 — Real stage tracking (built)

#### S3.1 — Move a paper to a real stage / تحريك بحث لمرحلة فعلية
- [ ] سجّل دخول كمحرّر → افتح بحث → اضغط "تغيير الحالة".
- [ ] **متوقع:** تظهر قائمة "الانتقال إلى مرحلة" فيها مراحل **هذا المركز** فقط.
- [ ] اختر مرحلة + احفظ → حقل "المرحلة الحالية" يتحدّث فوراً للمرحلة المختارة.

#### S3.2 — Audit trail / سجل التتبّع
- [ ] بعد التحريك، افتح "سجل المراحل" في نفس البحث.
- [ ] **متوقع:** يظهر سطر جديد فيه **اسم المرحلة** + **مين سوّاها** + الوقت + الملاحظات.

#### S3.3 — Status-only still works / تغيير الحالة بدون مرحلة
- [ ] غيّر حالة بحث **بدون** اختيار مرحلة → يشتغل عادي، والمرحلة الحالية ما تتغيّر/تنمسح.
- [ ] في مركز ما عنده مراحل مضبوطة → تظهر رسالة "لا مراحل" وتغيير الحالة يشتغل.

#### S3.4 — Non-editor can't move stages / غير المحرّر
- [ ] بحساب باحث/محكّم → ما يشوف زر تغيير الحالة/المرحلة إطلاقاً.

### Feature 1 — Committee voting

> ⚠️ يحتاج تطبيق الـ migration `20260619230000_committee_voting.sql` على Supabase أول.
> جهّز: محرّر + لجنة فيها عضوين أو أكثر (committee_member) في نفس المركز.

#### S3.5 — Assign a paper to a committee / إرسال بحث للجنة
- [ ] كمحرّر، افتح بحث → "إرسال للجنة" → اختر لجنة من نفس المركز → أرسل.
- [ ] **متوقع:** يتسجّل البحث للجنة. إعادة إرساله لنفس اللجنة → رسالة "مُسند مسبقاً".

#### S3.6 — Member votes with justification / تصويت العضو
- [ ] بحساب عضو لجنة، افتح البحث → يظهر صندوق التصويت بـ**4 خيارات**: موافقة / موافقة مع تعديلات / رفض / امتناع.
- [ ] جرّب الإرسال **بدون مبرّر** → الزر معطّل. اكتب مبرّر + صوّت → ينحفظ.
- [ ] **غيّر صوتك** ثم أعد الإرسال → يتحدّث (ما يضيف صوت ثاني). [هذا يثبت سياسة الـ UPDATE]

#### S3.7 — Tally is correct for a member / العدّاد صحيح للعضو
- [ ] بحساب عضو (مو محرّر)، تأكد إن العدّاد يبيّن **مجموع كل الأصوات** (مو "1/N" عالق). [هذا يثبت إصلاح الـ RPC]

#### S3.8 — Decision rules / قواعد القرار
- [ ] **الحد الأدنى (min votes):** قبل اكتمال النصاب → النتيجة "معلّق".
- [ ] **الأغلبية مع الامتناع:** مثال 2 موافق / 1 رفض / 2 امتناع → **معلّق** (الامتناع يصعّب التمرير، ما توصل أغلبية).
- [ ] **موافقة مع تعديلات تُحسب كموافقة:** 1 موافق + 2 موافقة-مع-تعديلات مقابل 1 رفض → **موافَق**.
- [ ] **إجماع + امتناع:** لو الكل صوّت وفيه امتناع/رفض → **مرفوض**.

#### S3.9 — Record decision / تسجيل القرار
- [ ] كمحرّر، لما تكتمل القاعدة → زر "تسجيل القرار" يشتغل → يتسجّل (موافَق/مرفوض) ويظهر في لوحة واضحة + يتسجّل في سجل مراحل البحث.

#### S3.10 — Vote privacy & walls / خصوصية الأصوات والجدران
- [ ] العضو العادي يشوف **صوته فقط** (مو مبرّرات الباقين). المحرّر يشوف **كل الأصوات والمبرّرات**.
- [ ] عضو من مركز ثاني ما يشوف/ما يصوّت على البحث.

#### S3.11 — Inbox queue / طابور صندوق المهام
- [ ] عضو لجنة عنده بحث ينتظر صوته → يظهر قسم "أبحاث بانتظار تصويتك" في صندوق المهام، والضغط يوديه للبحث. بعد ما يصوّت → يختفي من الطابور.
