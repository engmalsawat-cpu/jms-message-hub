# JMS — Test Scenarios / سيناريوهات الاختبار

> **ليش هذا الملف؟** هذا دليل اختبار يدوي **شامل** لكل النظام (JMS Message Hub):
> كل صفحة، كل زر، كل رابط، التسلسل الكامل للعمل، والصلاحيات لكل دور. نمشي عليه
> صفحة صفحة وسيناريو سيناريو.
>
> **Why this file?** A complete manual test plan covering the whole app — every
> page, button, link, the full paper lifecycle, and per-role permissions.
>
> **كيف نستخدمه:** كل سيناريو فيه: **الحساب المستخدم** + **الخطوات** + **النتيجة
> المتوقعة**. وقت الاختبار نعلّم `[x]` لو نجح، أو نكتب الخطأ لو فشل. أغلب هذي
> الاختبارات تنعمل في **Lovable preview** أو على المتصفح (مو في بيئة Claude).
>
> **الحسابات التجريبية (Test accounts)** — نبدّل بينها من **شريط "Test:" أعلى الصفحة**
> (RoleSwitcher) أو من أزرار "Quick Login" في صفحة الدخول. كل كلمات السر `123456789`:
>
> | الزر | البريد | الدور |
> |---|---|---|
> | Admin | `admin@test.com` | admin |
> | Editor | `editor@test.com` | editor_in_chief (رئيس تحرير) |
> | Managing | `managing@test.com` | managing_editor (مدير تحرير) |
> | Reviewer | `reviewer@test.com` | reviewer (محكّم) |
> | Researcher | `researcher@test.com` | researcher (باحث) |
> | Committee | `committee@test.com` | committee_member (عضو لجنة) |
> | HQ | `admin@test.com` | (نفس حساب Admin — انظر ملاحظة أدناه) |
>
> ⚠️ **ملاحظة مهمة عن HQ:** زر "HQ" في الدخول وفي شريط التبديل يسجّل دخول بنفس بريد
> Admin (`admin@test.com`)، فما فيه حساب `hq_admin` مستقل جاهز. لاختبار دور **مدير المقر
> (hq_admin)** لازم تضيف الدور يدوياً لمستخدم من صفحة المستخدمين، أو تستخدم `/seed-users`.

## 1. الدخول والصلاحيات (Authentication & Access)

### A1.1 — تسجيل الدخول بالبريد وكلمة السر / Login with credentials
- [ ] **الحساب:** أي حساب تجريبي. **خطوات:** افتح `/login` → اكتب البريد وكلمة السر → "تسجيل الدخول".
- [ ] **متوقع:** رسالة نجاح، ثم الانتقال إلى **`/dashboard`** (لوحة التحكم).
- [ ] **خطأ متوقع:** بريد/سر غلط → رسالة خطأ حمراء، وتبقى في صفحة الدخول.

### A1.2 — أزرار الدخول السريع / Quick Login buttons
- [ ] في `/login` تحت الفاصل تظهر أزرار ملوّنة (HQ / Admin / Editor / Managing / Reviewer / Researcher / Committee).
- [ ] اضغط أي زر → يدخل مباشرة بذلك الحساب وينتقل إلى `/dashboard`.
- [ ] **تحقّق:** زر **HQ** يدخل بنفس حساب Admin (نفس البريد) — مو حساب مستقل.

### A1.3 — إنشاء حساب جديد / Signup
- [ ] افتح `/signup` → عبّي الاسم الكامل + بريد + كلمة سر (٦ أحرف على الأقل) → "إنشاء حساب".
- [ ] **متوقع:** رسالة نجاح ثم الرجوع إلى **`/login`**.
- [ ] كلمة سر أقل من ٦ أحرف → المتصفح يمنع الإرسال (حقل minLength).
- [ ] روابط التنقل: "لديك حساب؟ تسجيل الدخول" يفتح `/login`، ومن الدخول "ليس لديك حساب؟" يفتح `/signup`.

### A1.4 — تسجيل الخروج / Logout
- [ ] من القائمة الجانبية (الأسفل) اضغط "تسجيل الخروج".
- [ ] **متوقع:** يخرج وتُمحى الجلسة. أي صفحة محمية تحوّلك لـ `/login`.

### A1.5 — حماية الصفحات (مستخدم غير مسجّل) / Protected routes redirect
- [ ] بدون تسجيل دخول، افتح أي رابط محمي مباشرة (مثلاً `/inbox` أو `/papers` أو `/users`).
- [ ] **متوقع:** تحويل فوري إلى **`/login`**.
- [ ] أثناء تحميل الجلسة تظهر دائرة تحميل قبل اتخاذ القرار (ما يطلع خطأ).

### A1.6 — الجذر "/" يحوّل لصندوق المهام / Root redirects to Inbox
- [ ] افتح الرابط الجذر `/` وأنت مسجّل دخول.
- [ ] **متوقع:** تحويل إلى **`/inbox`** (صندوق المهام)، مو لوحة التحكم.

### A1.7 — شريط تبديل الأدوار / Role switcher
- [ ] من شريط "Test:" أعلى الصفحة اضغط دوراً مختلفاً (مثلاً Reviewer).
- [ ] **متوقع:** يخرج من الحساب الحالي، يدخل بالحساب الجديد، رسالة "Switched to ..."، ثم تُحدّث الصفحة كاملة (reload).

## 2. التنقل والقائمة (Navigation & Layout)

> القائمة الجانبية مقسّمة 3 مجموعات حسب الدور:
> - **المجموعة الرئيسية (الكل):** صندوق المهام، لوحة التحكم، أبحاثي، تقديم بحث، الإشعارات، الملف الشخصي، لوحة المحكم.
> - **مجموعة المحرّرين (editor):** تظهر لـ admin / editor_in_chief / managing_editor / hq_admin → لوحة التشغيل، الأبحاث، المراكز، اللجان، الرسائل.
> - **مجموعة الإدارة (admin):** تظهر لـ admin / hq_admin فقط → المستخدمون، المالية، القائمة السوداء.

### N2.1 — القائمة لحساب باحث/محكّم / Sidebar for researcher & reviewer
- [ ] **الحساب:** Researcher (ثم Reviewer). افتح القائمة الجانبية.
- [ ] **متوقع:** تظهر **المجموعة الرئيسية فقط**؛ ما تظهر مجموعة المحرّرين ولا الإدارة.

### N2.2 — القائمة لحساب مدير التحرير / Sidebar for managing_editor
- [ ] **الحساب:** Managing. **متوقع:** تظهر المجموعة الرئيسية + **مجموعة المحرّرين** (لوحة التشغيل/الأبحاث/المراكز/اللجان/الرسائل)، لكن **لا تظهر** مجموعة الإدارة.

### N2.3 — القائمة لحساب رئيس التحرير / Sidebar for editor_in_chief
- [ ] **الحساب:** Editor. **متوقع:** نفس Managing (رئيسية + محرّرين)، **بدون** مجموعة الإدارة.

### N2.4 — القائمة لحساب Admin / hq_admin / Sidebar for admin
- [ ] **الحساب:** Admin. **متوقع:** تظهر **كل المجموعات الثلاث** بما فيها الإدارة (المستخدمون/المالية/القائمة السوداء).

### N2.5 — كل رابط يفتح الصفحة الصحيحة / Every nav link opens the right page
- [ ] اضغط كل عنصر في القائمة وتأكد من الوجهة:
  - صندوق المهام → `/inbox` · لوحة التحكم → `/dashboard` · أبحاثي → `/my-papers` · تقديم بحث → `/submit-paper` · الإشعارات → `/notifications` · الملف الشخصي → `/profile` · لوحة المحكم → `/reviewer`.
  - لوحة التشغيل → `/operations` · الأبحاث → `/papers` · المراكز → `/journals` · اللجان → `/committees` · الرسائل → `/messages`.
  - المستخدمون → `/users` · المالية → `/financial` · القائمة السوداء → `/blacklist`.
- [ ] العنصر النشط يتلوّن (خلفية مميّزة + لون primary).

### N2.6 — رأس الصفحة: زر الطيّ + اللغة + الجرس / Header controls
- [ ] زر طيّ القائمة (SidebarTrigger) يطوي/يفتح الشريط الجانبي (في الطيّ تبقى الأيقونات فقط).
- [ ] أيقونة الجرس أعلى اليمين تفتح `/notifications`، وعليها **شارة حمراء بعدد الإشعارات غير المقروءة** (تختفي لو صفر).
- [ ] زر اللغة (AR/EN) موجود في الرأس.

### N2.7 — رابط خاطئ يفتح 404 / NotFound for bad URLs
- [ ] افتح رابطاً غير موجود (مثلاً `/xyz123`).
- [ ] **متوقع:** صفحة **404 "Page not found"** فيها رابط "Return to Home".

## 3. سيناريوهات كل صفحة (Per-page Scenarios)

### 3.1 صندوق المهام (Inbox `/inbox`)
الغرض: يجمع المهام المعلّقة للمستخدم الحالي في 5 أقسام. (صفحة البداية بعد فتح "/").

- [ ] **P-INB.1 — أقسام تظهر فقط لو فيها عناصر:** كل قسم يظهر فقط إذا فيه بيانات؛ والقسم يحمل **شارة بعدد عناصره**.
- [ ] **P-INB.2 — طلبات تحكيم بانتظار ردك (Review requests):** بحساب Reviewer عنده طلب `pending` → يظهر القسم، والضغط على العنصر يوديه إلى **`/reviewer`**.
- [ ] **P-INB.3 — أبحاث تحت تحكيمك (Reviewing):** Reviewer قبِل تحكيماً (`accepted`) → يظهر، والضغط يفتح **`/review/{requestId}`** (نموذج التقرير).
- [ ] **P-INB.4 — أبحاثي تحتاج تعديلاً (Needs revision):** Researcher عنده بحث `revision_required` → يظهر، والضغط يفتح **`/papers/{id}`**.
- [ ] **P-INB.5 — أبحاث بانتظار تصويتك (Committee vote):** Committee عضو لجنة وعنده بحث محال للجنته ولم يصوّت بعد → يظهر، والضغط يفتح **`/papers/{id}`**. بعد التصويت يختفي العنصر. *(يعتمد على migration التصويت)*.
- [ ] **P-INB.6 — إشعارات غير مقروءة (Unread):** يعرض أحدث 5 إشعارات غير مقروءة، و"عرض الكل" يفتح **`/notifications`**.
- [ ] **P-INB.7 — الحالة الفارغة:** بحساب ما عنده أي مهام → كل الأقسام تختفي وتظهر رسالة واحدة **"لا توجد مهام معلقة حالياً 🎉"**.
- [ ] **P-INB.8 — RTL/LTR:** عربي = الصفحة يمين←يسار، إنجليزي = يسار←يمين.

### 3.2 لوحة التحكم (Dashboard `/dashboard`)
الغرض: بطاقات إحصائية + ترحيب + قائمة أدوارك.

- [ ] **P-DASH.1 — الترحيب:** يظهر "مرحباً، {اسمك}".
- [ ] **P-DASH.2 — بطاقات الكل:** بطاقتا "أبحاثي" (عدد أبحاثك) و"الإشعارات" (غير المقروءة) تظهران لكل المستخدمين. الضغط عليهما يفتح `/my-papers` و`/notifications`.
- [ ] **P-DASH.3 — بطاقات المحرّر:** بحساب محرّر (admin/editor/managing/hq) تظهر بطاقات إضافية: الأبحاث (`/papers`)، المراكز (`/journals`)، اللجان (`/committees`) بأعدادها.
- [ ] **P-DASH.4 — بطاقات المحرّر تختفي لغير المحرّر:** بحساب Researcher/Reviewer لا تظهر بطاقات الأبحاث/المراكز/اللجان.
- [ ] **P-DASH.5 — بطاقة الأدوار:** تعرض كل أدوارك بأسمائها الصحيحة؛ ومن لا دور له يظهر "باحث".

### 3.3 أبحاثي (My Papers `/my-papers`)
الغرض: جدول أبحاث المستخدم الحالي فقط.

- [ ] **P-MYP.1 — العرض:** جدول بأعمدة (العنوان / المركز / الحالة / إجراء). يعرض أبحاث `submitted_by = أنا` فقط.
- [ ] **P-MYP.2 — زر "بحث جديد":** يفتح `/submit-paper`.
- [ ] **P-MYP.3 — أيقونة العين:** تفتح صفحة تفاصيل البحث `/papers/{id}`.
- [ ] **P-MYP.4 — شارة الحالة:** لون الشارة يطابق الحالة (submitted أزرق، accepted أخضر، rejected أحمر... إلخ).
- [ ] **P-MYP.5 — حالة فارغة:** حساب بلا أبحاث → "لا توجد بيانات".

### 3.4 تقديم بحث (Submit Paper `/submit-paper`)
الغرض: نموذج تقديم بحث جديد.

- [ ] **P-SUB.1 — قائمة المراكز:** القائمة المنسدلة تعرض المراكز **النشطة (active)** فقط.
- [ ] **P-SUB.2 — الحقول الثنائية:** عنوان/ملخص بالعربي (RTL) وبالإنجليزي (LTR)؛ العنوانان مطلوبان.
- [ ] **P-SUB.3 — الكلمات المفتاحية:** تُكتب مفصولة بفواصل وتُحفظ كقائمة.
- [ ] **P-SUB.4 — رفع الملف:** يقبل `.pdf/.doc/.docx`؛ بعد الاختيار تظهر أيقونة خضراء. (الملف اختياري).
- [ ] **P-SUB.5 — زر الإرسال معطّل بدون مركز:** الزر يبقى معطّلاً حتى تختار مركزاً.
- [ ] **P-SUB.6 — نجاح التقديم:** بعد الإرسال → رسالة "تم تقديم البحث بنجاح" والانتقال إلى **`/my-papers`**؛ والبحث الجديد بحالة **submitted** ويظهر في القائمة.

### 3.5 الأبحاث (Papers list `/papers`) — محرّرون
الغرض: كل الأبحاث (ضمن جدار المركز) مع بحث وفلترة.

- [ ] **P-PAP.1 — العرض:** جدول (العنوان / الباحث / المركز / الحالة / المرحلة / التاريخ / إجراء).
- [ ] **P-PAP.2 — البحث:** الكتابة في خانة البحث تُصفّي حسب العنوان (عربي/إنجليزي).
- [ ] **P-PAP.3 — فلتر الحالة:** اختيار حالة من القائمة يُصفّي الجدول؛ "الكل" يرجع كل شيء.
- [ ] **P-PAP.4 — بطاقات الملخص:** 4 بطاقات (submitted / under_review / accepted / published) بأعدادها، والضغط على بطاقة يضبط الفلتر على تلك الحالة.
- [ ] **P-PAP.5 — أيقونة العين:** تفتح `/papers/{id}`.
- [ ] **P-PAP.6 — وصول الدور:** الرابط `/papers` يظهر في القائمة للمحرّرين فقط (انظر قسم الأدوار).

### 3.6 تفاصيل البحث (Paper Detail `/papers/:id`)
الغرض: كل شيء عن بحث واحد + إجراءات المحرّر.

- [ ] **P-PD.1 — الرأس:** عنوان البحث + شارة الحالة + زر الرجوع (السهم). زر الرجوع يوديك إلى `/papers` للمحرّر، وإلى `/my-papers` لغيره. (في العربية السهم يتجه يميناً).
- [ ] **P-PD.2 — بطاقتا العنوان/الملخص:** عربي (RTL) وإنجليزي (LTR)، وبطاقة "معلومات البحث" (المركز/الكلمات/المرحلة الحالية/تاريخ التقديم).
- [ ] **P-PD.3 — تحميل الملف:** لو فيه ملف مرفوع، زر "تحميل الملف" يفتح رابطاً موقّعاً مؤقتاً (signed URL) في تبويب جديد.
- [ ] **P-PD.4 — أزرار المحرّر تظهر للمحرّر فقط:** "تغيير الحالة" / "تعيين محكّم" / "مراسلة الباحث" / "إرسال إلى لجنة" تظهر فقط لـ admin/editor/managing/hq.
- [ ] **P-PD.5 — تغيير الحالة:** افتح "تغيير الحالة" → اختر حالة جديدة (القائمة تستثني الحالة الحالية) + (اختياري) مرحلة من **مراحل هذا المركز فقط** + ملاحظات → "تأكيد".
  - **متوقع:** تتحدّث شارة الحالة فوراً؛ ولو اخترت مرحلة تتحدّث "المرحلة الحالية"؛ ويظهر سطر جديد في "سجل المراحل".
- [ ] **P-PD.6 — تغيير الحالة بدون مرحلة:** بدون اختيار مرحلة → يشتغل، والمرحلة الحالية ما تتغيّر.
- [ ] **P-PD.7 — مركز بلا مراحل:** لو المركز ما عنده مراحل تظهر رسالة "لا توجد مراحل معرّفة لهذا المركز" وتغيير الحالة يبقى ممكناً.
- [ ] **P-PD.8 — تعيين دور (محكّم/محرر):** "تعيين محكّم" → بريد + دور (reviewer/editor) → "تأكيد". بريد غير موجود → "لم يتم العثور على المستخدم". بعد النجاح يظهر في بطاقة "الأدوار المعينة".
- [ ] **P-PD.9 — مراسلة الباحث:** "مراسلة الباحث" → اكتب رسالة → "إرسال". تظهر في بطاقة "الرسائل" بالأسفل؛ والرسائل اللاحقة تُضاف لنفس المحادثة (thread واحد من نوع editor_author).
- [ ] **P-PD.10 — إرسال إلى لجنة:** "إرسال إلى لجنة" → اختر لجنة من **لجان نفس المركز** → "تأكيد" → "تم إرسال البحث إلى اللجنة". إعادة الإرسال لنفس اللجنة → "هذا البحث محال مسبقاً إلى هذه اللجنة". لو ما فيه لجان → "لا توجد لجان لهذا المركز". *(يعتمد على migration التصويت)*.
- [ ] **P-PD.11 — لوحة طلبات التحكيم (محرّر):** تظهر بطاقة "طلبات التحكيم" مع زر "إرسال طلب تحكيم" (بريد + تاريخ استحقاق + ملاحظات) وقائمة الطلبات بحالاتها، وزر إلغاء (X) للطلبات المعلّقة.
- [ ] **P-PD.12 — لوحة تصويت اللجنة:** تظهر لو البحث محال للجنة — للأعضاء والمحررين. (تفاصيلها في قسم اللجان والتسلسل).
- [ ] **P-PD.13 — تقارير التحكيم (محرّر):** بعد تقديم محكّم لتقريره، تظهر بطاقة "تقارير التحكيم المقدمة" بالدرجات والتوصية والملاحظات العامة والسرية (السرية للمحرّر فقط).
- [ ] **P-PD.14 — تنبيه التعديل للباحث:** بحساب الباحث صاحب البحث وحالته `revision_required` → يظهر صندوق برتقالي "مطلوب تعديلات".
- [ ] **P-PD.15 — سجل المراحل والتتبع (للجميع):** يعرض الأحداث بالترتيب الزمني، كل سطر فيه الوصف + اسم المرحلة (إن وُجدت) + الملاحظات + "بواسطة: {اسم}" + الوقت. آخر سطر مميّز بعلامة ✓.

### 3.7 المراكز (Centers / Journals `/journals`)
الغرض: عرض المراكز وإدارتها + إدارة مراحل سير العمل لكل مركز.

> 🔤 **تسمية:** في كل النظام نقول **"المراكز" (Centers)** مو "المجلات"؛ تأكّد من ذلك في القائمة الجانبية والعنوان وزر الإضافة والرسائل.

- [ ] **P-JRN.1 — العرض:** بطاقات المراكز (الاسم بالعربي/الإنجليزي + الوصف + ISSN + شارة الحالة active/inactive/archived).
- [ ] **P-JRN.2 — زر "مركز جديد" (إدارة):** يظهر فقط لـ admin/editor_in_chief/hq_admin. **لا يظهر** لمدير التحرير (managing).
- [ ] **P-JRN.3 — إنشاء مركز:** العنوانان مطلوبان (الزر معطّل بدونهما) → "إنشاء" → رسالة **"تم إنشاء المركز"** (EN: "Center created") ويظهر فوراً.
- [ ] **P-JRN.4 — مراحل سير العمل (WorkflowStagesManager):** في كل بطاقة قسم "مراحل سير العمل":
  - **تحميل المراحل الافتراضية:** مركز بلا مراحل → يظهر زر "تحميل المراحل الافتراضية". اضغطه → تنعبّي 5 مراحل بالترتيب (الفحص الأولي → التحكيم العلمي → تصويت اللجنة → القرار النهائي → النشر) ورسالة نجاح، **ويختفي الزر** بعد ذلك.
  - **إضافة مرحلة:** "إضافة مرحلة" (اسم عربي/إنجليزي + نوع screening/review/publication).
  - **تعديل/حذف/ترتيب:** أزرار التعديل (قلم)، الحذف (سلة، مع تأكيد)، والأسهم لأعلى/أسفل لإعادة الترتيب.
  - **غير المحرّر:** أزرار الإضافة/التعديل/الترتيب/الحذف وزر التحميل **لا تظهر** (عرض فقط).

### 3.8 اللجان (Committees `/committees`)
الغرض: عرض اللجان وإدارتها + أعضائها + الأبحاث المحالة ونتائج التصويت.

- [ ] **P-CMT.1 — العرض:** بطاقات لجان (الاسم + المركز + النوع + آلية التصويت + الحد الأدنى للأصوات + الأعضاء + الأبحاث المحالة).
- [ ] **P-CMT.2 — إنشاء لجنة (إدارة):** زر "لجنة جديدة" يظهر لـ admin/editor_in_chief/hq_admin فقط (**ليس** managing). الحقول: اسم عربي/إنجليزي + المركز + آلية (أغلبية/إجماع/موزون) + الحد الأدنى. الزر معطّل حتى تكتمل الحقول المطلوبة.
- [ ] **P-CMT.3 — إضافة عضو:** "إضافة عضو" → بريد + (اختياري) رئيس اللجنة → "تأكيد". بريد غير موجود → "لم يتم العثور على المستخدم". يظهر العضو في القائمة، ورئيس اللجنة بشارة "رئيس".
- [ ] **P-CMT.4 — إزالة عضو:** أيقونة السلة بجانب العضو (للمحرّر) تزيله مع رسالة.
- [ ] **P-CMT.5 — الأبحاث المحالة والعدّاد:** لكل بحث محال تظهر شارة: إما القرار النهائي (موافقة/رفض) لو محسوم، أو عدّاد `أصوات/أعضاء`. *(يعتمد على migration التصويت)*.
- [ ] **P-CMT.6 — حالة فارغة:** بلا لجان → "لا توجد بيانات".

### 3.9 الرسائل (Messages `/messages`)
الغرض: محادثات (threads) المرتبطة بالأبحاث.

- [ ] **P-MSG.1 — العرض:** قائمة المحادثات (العنوان + عدد الرسائل + التاريخ + نوع المحادثة). تُرتّب بالأحدث.
- [ ] **P-MSG.2 — التوسيع:** الضغط على المحادثة يفتح/يطوي الرسائل؛ رسائلي بمحاذاة جهة ورسائل غيري بالجهة الأخرى.
- [ ] **P-MSG.3 — الرد:** اكتب رداً → زر الإرسال → "تم إرسال الرد" وتظهر الرسالة. (الزر معطّل لو الحقل فارغ).
- [ ] **P-MSG.4 — حالة فارغة:** بلا محادثات → رسالة "لا توجد رسائل حالياً" مع شرح. *(الصفحة تعتمد على RLS لتقييد المحادثات للمشاركين فيها — تحقّق أن المستخدم يرى محادثاته فقط).*

### 3.10 الإشعارات (Notifications `/notifications`)
الغرض: إشعارات المستخدم الحالي.

- [ ] **P-NOT.1 — العرض:** قائمة إشعارات (عنوان + نص + تاريخ) للمستخدم الحالي، الأحدث أولاً. المقروء يظهر باهتاً.
- [ ] **P-NOT.2 — تعليم كمقروء:** زر ✓ على الإشعار غير المقروء → يصير مقروءاً، وتنقص شارة الجرس في الرأس.
- [ ] **P-NOT.3 — حالة فارغة:** بلا إشعارات → "لا توجد بيانات". *(الإشعارات تُنشأ من قاعدة البيانات/المشغّلات، لا يوجد زر إنشاء يدوي في الواجهة).*

### 3.11 المستخدمون (Users `/users`)
الغرض: قائمة المستخدمين وإدارة أدوارهم (للأدمن).

- [ ] **P-USR.1 — العرض:** جدول (الاسم / البريد / الأدوار / تاريخ التسجيل). يظهر عمود "إجراءات" للأدمن فقط (admin/hq_admin).
- [ ] **P-USR.2 — البحث:** يُصفّي حسب الاسم أو البريد.
- [ ] **P-USR.3 — إضافة دور (أدمن):** أيقونة + → اختر دوراً من **الأدوار السبعة** → "تأكيد" → يُضاف ويظهر كشارة.
- [ ] **P-USR.4 — إزالة دور (أدمن):** أيقونة السلة داخل شارة الدور تزيله.
- [ ] **P-USR.5 — أسماء الأدوار:** كل الأدوار السبعة تظهر بأسماء صحيحة عربي/إنجليزي. مستخدم بلا أدوار يظهر "باحث".
- [ ] **P-USR.6 — غير الأدمن:** بحساب editor/managing يرى الجدول لكن **بدون** عمود/أزرار إدارة الأدوار.

### 3.12 المالية (Financial `/financial`)
الغرض: سجلات مالية (عرض فقط).

- [ ] **P-FIN.1 — البطاقات العلوية:** إجمالي المدفوعات (paid) + قيد الانتظار (pending) + الإجمالي، محسوبة من السجلات.
- [ ] **P-FIN.2 — الجدول:** (المركز / البحث / النوع / المبلغ+العملة / الحالة) مع شارات ملوّنة (paid/pending/overdue).
- [ ] **P-FIN.3 — وصول الدور:** الرابط يظهر في القائمة للأدمن فقط. *(الصفحة نفسها بلا حارس دور — RLS هي اللي تحمي البيانات؛ لو فُتحت بالرابط المباشر بحساب غير أدمن، تأكّد أنها لا تكشف بيانات مراكز أخرى).*

### 3.13 القائمة السوداء (Blacklist `/blacklist`)
الغرض: قائمة بريد ممنوع لكل مركز.

- [ ] **P-BLK.1 — العرض:** جدول (البريد / المركز / السبب / التاريخ / إجراء).
- [ ] **P-BLK.2 — إضافة:** زر "إضافة" → بريد + مركز + سبب → "تأكيد" → يُضاف. (البريد والمركز مطلوبان).
- [ ] **P-BLK.3 — إزالة:** أيقونة السلة تزيل الإدخال.
- [ ] **P-BLK.4 — حالة فارغة:** بلا إدخالات → "لا توجد بيانات".
- [ ] **P-BLK.5 — وصول الدور:** الرابط في القائمة للأدمن فقط. *(زر "إضافة" يظهر داخل الصفحة لأي مستخدم يفتح الرابط؛ RLS تتكفّل بمنع الكتابة عبر المراكز — تحقّق من ذلك).*

### 3.14 لوحة التشغيل (Operations `/operations`) — محرّرون
الغرض: تحليلات + كانبان + المتأخرات + المحكمون + تصدير CSV.

- [ ] **P-OPS.1 — العنوان والإحصائيات:** عدد الأبحاث الكلي + 6 بطاقات حالة سريعة (الضغط عليها يفلتر).
- [ ] **P-OPS.2 — تبويب التحليلات:** 4 رسوم (دائري حسب الحالة / خطي شهري / أعمدة حسب المركز / نسب القبول والرفض) تظهر صحيحة.
- [ ] **P-OPS.3 — تبويب كانبان:** أعمدة حسب الحالة؛ بحث + فلتر مركز؛ كل بطاقة تفتح `/papers/{id}` بالضغط؛ تظهر أيام التأخر >7 أيام بعلامة. أعمدة فارغة تقول "لا توجد أبحاث".
- [ ] **P-OPS.4 — تبويب المتأخرة:** اختيار المدة (7/14/30/60 يوم) يحدّث قائمة الأبحاث غير المحدّثة (الحالات النشطة فقط) + شارة العدد؛ "العين" تفتح التفاصيل. بلا متأخرات → "لا توجد أبحاث متأخرة 🎉".
- [ ] **P-OPS.5 — تبويب المحكمون:** رسم أعمدة + جدول عدد الأبحاث لكل محكّم. بلا محكمين → "لا يوجد محكمون معينون".
- [ ] **P-OPS.6 — تصدير CSV:** زر "تصدير CSV" ينزّل ملفاً يشمل الأبحاث المفلترة، بترميز يدعم العربية (BOM).

### 3.15 الملف الشخصي (Profile `/profile`)
الغرض: بيانات المستخدم + المؤهلات + الإنتاج العلمي + الخبرات (CRUD).

- [ ] **P-PRO.1 — تبويب المعلومات:** تعديل (الاسم/الهاتف/الرتبة الأكاديمية/الجهة/الدولة/ORCID/Google Scholar/نبذة) → "حفظ" → "تم تحديث الملف".
- [ ] **P-PRO.2 — المؤهلات:** "إضافة" (الدرجة/التخصص/الجهة/سنة التخرج/الدولة) → حفظ → يظهر. تعديل (قلم) وحذف (سلة) يعملان.
- [ ] **P-PRO.3 — الإنتاج العلمي:** إضافة/تعديل/حذف (النوع/العنوان/الناشر/التاريخ/الرابط/الوصف).
- [ ] **P-PRO.4 — الخبرات:** إضافة/تعديل/حذف (المسمى/المنظمة/تاريخ البداية/النهاية/حالي؟/الوصف).
- [ ] **P-PRO.5 — حالة فارغة:** كل قسم بلا عناصر يقول "لا توجد بيانات".

### 3.16 لوحة المحكم (Reviewer Dashboard `/reviewer`)
الغرض: طلبات التحكيم الخاصة بالمحكّم (قبول/رفض + الدخول للتقرير).

- [ ] **P-RVD.1 — الإحصائيات:** 3 بطاقات (معلقة / جارٍ / مكتمل) بأعدادها.
- [ ] **P-RVD.2 — قبول طلب:** في "طلبات تحكيم جديدة" زر "قبول" → يصير الطلب `accepted` وينتقل لقسم "تحكيم جارٍ".
- [ ] **P-RVD.3 — رفض طلب:** زر "رفض" أول ضغطة تُظهر خانة ملاحظات؛ الضغطة الثانية تسجّل `declined` مع الملاحظات.
- [ ] **P-RVD.4 — عرض الملف:** في "تحكيم جارٍ" زر "عرض الملف" يفتح ملف البحث (signed URL).
- [ ] **P-RVD.5 — تعبئة التقرير:** زر "تعبئة التقرير" يفتح `/review/{requestId}`.
- [ ] **P-RVD.6 — حالة فارغة:** بلا طلبات → "لا توجد طلبات تحكيم".

### 3.17 نموذج التحكيم (Review Form `/review/:requestId`)
الغرض: تقييم البحث حسب معايير المركز + توصية + ملاحظات.

- [ ] **P-RVF.1 — رأس + ملخص + ملف:** عنوان البحث + ملخصه + زر تحميل الملف.
- [ ] **P-RVF.2 — معايير التقييم:** تظهر معايير **هذا المركز** (active) كل واحد بمنزلق درجات (0..max) + تعليق. (لو ما فيه معايير، القسم لا يظهر).
- [ ] **P-RVF.3 — التوصية والملاحظات:** اختيار توصية (قبول/تعديلات طفيفة/تعديلات جوهرية/رفض) + ملاحظات عامة (تظهر للباحث) + ملاحظات سرية (للمحرّر فقط).
- [ ] **P-RVF.4 — حفظ كمسودة:** "حفظ كمسودة" → "تم حفظ المسودة" بدون مغادرة؛ ولا يُحسب التقرير مُقدّماً.
- [ ] **P-RVF.5 — إرسال التقرير:** بدون اختيار توصية → خطأ "يرجى اختيار التوصية". مع توصية → يُرسل، يصير الطلب `completed`، رسالة نجاح، والرجوع إلى `/reviewer`.
- [ ] **P-RVF.6 — استرجاع مسودة:** إعادة فتح النموذج تعرض القيم المحفوظة سابقاً.
- [ ] **P-RVF.7 — زر الرجوع:** السهم يرجع إلى `/reviewer` (اتجاه السهم يتبع اللغة).

## 4. التسلسل الكامل للعمل (End-to-End Workflows)

> هذه رحلات تمر عبر عدة صفحات وعدة أدوار. بدّل الأدوار من شريط "Test:".
> جهّز مسبقاً: مركز فيه مراحل (حمّل الافتراضية)، ولجنة فيها عضوان على الأقل في نفس المركز.

### W4.1 — دورة حياة البحث الكاملة / Full lifecycle
1. [ ] **Researcher:** `/submit-paper` → قدّم بحثاً في مركز فعّال → الحالة **submitted** ويظهر في `/my-papers`.
2. [ ] **Editor:** `/papers` → افتح البحث → "تغيير الحالة" إلى **under_review** (+ مرحلة "التحكيم العلمي") → الحالة والمرحلة تتحدّثان ويُسجّل في السجل.
3. [ ] **Editor:** في تفاصيل البحث، "طلبات التحكيم" → "إرسال طلب تحكيم" ببريد `reviewer@test.com` → الطلب يظهر **pending**.
4. [ ] **Reviewer:** `/inbox` أو `/reviewer` → يرى الطلب → "قبول" → يصير **accepted** وينتقل لـ "تحكيم جارٍ".
5. [ ] **Reviewer:** "تعبئة التقرير" → عبّي الدرجات + توصية + ملاحظات → "إرسال التقرير" → الطلب **completed**.
6. [ ] **Editor:** تفاصيل البحث → تظهر بطاقة "تقارير التحكيم المقدمة" بالدرجات والتوصية والملاحظات السرية.
7. [ ] **Editor:** "إرسال إلى لجنة" → اختر لجنة المركز → "تأكيد".
8. [ ] **Committee (لكل عضو):** `/inbox` "أبحاث بانتظار تصويتك" → افتح البحث → صوّت (أحد الخيارات الأربعة) **مع مبرّر** → ينحفظ ويختفي من طابور المهام.
9. [ ] **Editor:** لما يكتمل النصاب وتتحدّد النتيجة → يظهر زر "تسجيل القرار: موافقة/رفض" → اضغطه → القرار يُسجّل في لوحة اللجنة و**يُكتب سطر "قرار اللجنة" في سجل مراحل البحث**.
10. [ ] **Editor:** "تغيير الحالة" إلى **accepted** ثم **published** (+ مرحلة "النشر") → يظهر التدرّج في السجل وفي `/papers` و`/operations`.

### W4.2 — حلقة التعديل / Revision loop
1. [ ] **Editor:** غيّر حالة بحث إلى **revision_required** (مع ملاحظات).
2. [ ] **Researcher:** `/inbox` "أبحاثي تحتاج تعديلاً" أو `/papers/{id}` → يظهر الصندوق البرتقالي "مطلوب تعديلات".
3. [ ] **Editor (بعد رد الباحث):** غيّر الحالة إلى **revised** ثم تابع الدورة. تأكّد أن كل تغيير يُسجّل في السجل.

### W4.3 — المراسلة بين المحرّر والباحث / Editor↔Author messaging
1. [ ] **Editor:** `/papers/{id}` → "مراسلة الباحث" → أرسل رسالة → تظهر في بطاقة "الرسائل".
2. [ ] **Researcher:** `/messages` → يرى نفس المحادثة → يرد عليها → الرد يظهر للطرفين على نفس المحادثة.
3. [ ] رسائل المحرّر اللاحقة تبقى في **نفس** المحادثة (لا تنشئ محادثة جديدة).

### W4.4 — توليد الإشعارات / Notifications along the way
- [ ] خلال الخطوات أعلاه، راقب شارة الجرس و`/inbox`/`/notifications`: لو فعّلت قاعدة البيانات مشغّلات الإشعارات، تظهر إشعارات عند الأحداث (طلب تحكيم/قرار/تعديل...). علّم الإشعار كمقروء وتأكّد أن الشارة تنقص. *(لو ما توجد مشغّلات، الإشعارات قد لا تتولّد تلقائياً — سجّل ذلك).*

### W4.5 — قواعد قرار اللجنة / Committee decision rules
*(يعتمد على migration `committee_voting`)*
- [ ] **النصاب (min_votes):** قبل اكتمال الحد الأدنى → النتيجة **"معلّق"** بغضّ النظر عن الأصوات.
- [ ] **أغلبية مع امتناع:** 2 موافقة / 1 رفض / 2 امتناع → **معلّق** (الامتناع داخل المقام، فما تتحقّق الأغلبية الصارمة).
- [ ] **"موافقة مع تعديلات" تُحسب موافقة:** 1 موافقة + 2 (موافقة مع تعديلات) مقابل 1 رفض → **موافَق**.
- [ ] **إجماع (unanimous):** يصير "موافَق" فقط لو صوّت **كل** الأعضاء وكلها إيجابية؛ وجود رفض أو امتناع بعد اكتمال الجميع → **مرفوض**.
- [ ] **تغيير الصوت:** العضو يغيّر صوته ويعيد الإرسال → يُحدَّث (لا يُضاف صوت ثانٍ).
- [ ] **زر التصويت معطّل بلا مبرّر:** ما تقدر ترسل صوتاً بدون كتابة المبرّر.
- [ ] **العدّاد صحيح للعضو:** العضو العادي يرى المجاميع الصحيحة عبر الـ RPC (مو عالقة على صوته فقط).
- [ ] **الخصوصية:** العضو يرى **صوته فقط**؛ المحرّر يرى **كل الأصوات والمبرّرات**. عضو من مركز آخر لا يرى/لا يصوّت.

## 5. الأدوار والصلاحيات (Roles & Permissions Matrix)

> القاعدة العامة: "محرّر (editor group)" = admin + editor_in_chief + managing_editor + hq_admin.
> لكن **إدارة المراكز واللجان والمراحل** مقيّدة بـ admin + editor_in_chief + hq_admin **فقط** (بدون managing_editor).
> و**مجموعة الإدارة** (المستخدمون/المالية/القائمة السوداء) في القائمة لـ admin + hq_admin فقط.

### R5.1 — باحث (researcher)
- [ ] **يرى:** صندوق المهام، لوحة التحكم (بدون بطاقات المحرّر)، أبحاثي، تقديم بحث، الإشعارات، الملف، لوحة المحكم.
- [ ] **لا يرى:** مجموعتي المحرّرين والإدارة في القائمة؛ ولا أزرار المحرّر في تفاصيل البحث.

### R5.2 — محكّم (reviewer)
- [ ] مثل الباحث في القائمة، لكنه يستقبل طلبات تحكيم في `/inbox` و`/reviewer` ويعبّي تقارير في `/review/:id`.
- [ ] **لا يرى:** أزرار إدارة الأبحاث/المراكز/اللجان.

### R5.3 — عضو لجنة (committee_member)
- [ ] مثل الباحث في القائمة، لكنه يرى قسم "أبحاث بانتظار تصويتك" ويصوّت في لوحة تصويت اللجنة داخل تفاصيل البحث (لجانه فقط).
- [ ] **لا يرى:** أصوات غيره ولا إدارة اللجان.

### R5.4 — مدير التحرير (managing_editor)
- [ ] **يرى:** المجموعة الرئيسية + مجموعة المحرّرين (لوحة التشغيل/الأبحاث/المراكز/اللجان/الرسائل).
- [ ] **يستطيع:** تغيير حالة الأبحاث، تعيين محكّمين، المراسلة، الإرسال للجان، ورؤية تصويت اللجان.
- [ ] **لا يستطيع:** إنشاء/إدارة المراكز (زر "مركز جديد" مخفي)، إنشاء/إدارة اللجان (زر "لجنة جديدة" مخفي)، إدارة مراحل سير العمل (أزرار الإضافة/التعديل/الحذف/التحميل مخفية).
- [ ] **لا يرى:** مجموعة الإدارة (المستخدمون/المالية/القائمة السوداء) في القائمة.

### R5.5 — رئيس التحرير (editor_in_chief)
- [ ] كل ما يستطيعه مدير التحرير، **إضافةً إلى** إدارة المراكز واللجان ومراحل سير العمل (الأزرار تظهر له).
- [ ] **لا يرى:** مجموعة الإدارة في القائمة.

### R5.6 — مدير النظام (admin)
- [ ] يرى **كل** القائمة (الرئيسية + المحرّرين + الإدارة)، ويدير المستخدمين/الأدوار والمالية والقائمة السوداء، وكل ما سبق.

### R5.7 — مدير المقر (hq_admin)
- [ ] مثل admin في القائمة (الكل) وإدارة المراكز/اللجان/المراحل.
- [ ] **مميّز:** يرى بيانات **كل المراكز** مجمّعة (يتجاوز جدران المراكز). انظر القسم 6.
- [ ] ⚠️ **تجهيز:** لا يوجد حساب hq_admin جاهز؛ أضف الدور `hq_admin` لمستخدم من `/users` لاختبار هذه الحالة.

## 6. جدران المراكز / الأمان (Center Walls / Security)

> ⚠️ تُختبر على **قاعدة البيانات الحيّة بعد تطبيق migrations** (مو في المعاينة الفارغة).
> جهّز: **U_A** (عضو مركز A فقط)، **U_B** (عضو مركز B فقط)، **U_HQ** (`hq_admin`).

- [ ] **S6.1 — جدار الأبحاث:** بـ U_A، `/papers` يعرض أبحاث مركز A فقط؛ U_B يرى B فقط.
- [ ] **S6.2 — مدير المقر يرى الكل:** U_HQ يرى أبحاث/لجان/مالية كل المراكز مجمّعة.
- [ ] **S6.3 — المنع بالرابط المباشر:** بـ U_A، فتح `/papers/{id}` لبحث من مركز B → لا بيانات (محظور فعلياً، مو مخفي فقط).
- [ ] **S6.4 — جدران على كل الأنواع:** بـ U_A، يرى بيانات مركزه فقط في: اللجان، المالية، القائمة السوداء، نوافذ التقديم، معايير التقييم، الرسائل، تقارير التحكيم.
- [ ] **S6.5 — لا أحد منقفل:** الحسابات الحقيقية الموجودة تشوف بياناتها (مو شاشة فاضية). لو محرّر عام بلا عضوية يرى فراغاً، أضف له عضوية أو `hq_admin`.
- [ ] **S6.6 — منع الكتابة عبر المراكز:** بـ U_A، محاولة إنشاء بحث/لجنة وربطه بمركز B → يُرفض.
- [ ] **S6.7 — لا انحدار:** الباحث يرى أبحاثه، المحكّم يرى المسند له، الإشعارات تظهر كالمعتاد.

## 7. الترابط بين الصفحات (Cross-page Linkage & Integrity)

- [ ] **L7.1 — من صندوق المهام:** كل عنصر يفتح وجهته الصحيحة (طلب تحكيم → `/reviewer` · تحكيم جارٍ → `/review/:id` · تعديل مطلوب → `/papers/:id` · تصويت لجنة → `/papers/:id` · "عرض الكل" → `/notifications`).
- [ ] **L7.2 — من الجرس/الإشعارات:** الجرس يفتح `/notifications`؛ والشارة تطابق عدد غير المقروء وتنقص عند التعليم كمقروء.
- [ ] **L7.3 — من قوائم الأبحاث:** "العين" في `/my-papers` و`/papers`، وبطاقات كانبان والمتأخرات في `/operations` كلها تفتح `/papers/:id` الصحيح.
- [ ] **L7.4 — الرجوع:** زر الرجوع في تفاصيل البحث يوديك إلى `/papers` (محرّر) أو `/my-papers` (غيره)؛ وفي نموذج التحكيم إلى `/reviewer`.
- [ ] **L7.5 — انعكاس البيانات:** تغيير الحالة/المرحلة في تفاصيل البحث ينعكس في `/papers` و`/operations`؛ تسجيل قرار اللجنة يظهر في `/committees` (شارة القرار) و**في سجل مراحل** نفس البحث.
- [ ] **L7.6 — المراسلة:** رسالة من تفاصيل البحث تظهر في `/messages` للطرفين، والرد يظهر في الموضعين.
- [ ] **L7.7 — تقارير التحكيم:** تقرير مُرسل من `/review/:id` يظهر في بطاقة "تقارير التحكيم المقدمة" داخل تفاصيل البحث للمحرّر.

## 8. اللغة والاتجاه (Bilingual & RTL)

- [ ] **B8.1 — التبديل الفوري:** زر اللغة (في الرأس وفي صفحات الدخول/التسجيل) يبدّل AR/EN وكل النصوص تتغيّر فوراً.
- [ ] **B8.2 — الاتجاه:** عربي = RTL (والقائمة الجانبية تنتقل لليمين)؛ إنجليزي = LTR (القائمة لليسار). الأسهم (الرجوع) تنعكس مع اللغة.
- [ ] **B8.3 — ثبات الاختيار:** اللغة محفوظة (localStorage) وتبقى بعد إعادة التحميل.
- [ ] **B8.4 — لا نصوص مكسورة:** افتح **كل صفحة** بكل لغة وتأكّد: لا مفاتيح ترجمة ظاهرة (مثل `nav.inbox`)، ولا نصوص ناقصة، والعناوين الثنائية (عربي/إنجليزي) في الأماكن الصحيحة.
- [ ] **B8.5 — التسمية "المراكز":** في العربية "المراكز" وفي الإنجليزية "Centers" (مو "Journals/المجلات") في القائمة والعناوين والرسائل.
